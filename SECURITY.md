# Security Policy / 安全策略

## Supported Versions / 支持版本

| Version | Supported |
| --- | --- |
| `v1.2.x` | Yes |
| `< v1.2.0` | No |

当前优先维护 `v1.2.x` 分支中的安全问题修复。  
Security fixes are prioritized for the `v1.2.x` line.

## Reporting a Vulnerability / 漏洞上报

Please use GitHub Security Advisories for private vulnerability reporting:  
请优先通过 GitHub Security 私密渠道提交漏洞报告：

`https://github.com/EagleFandel/metrics-agent/security/advisories/new`

For non-sensitive questions, you can open a public issue:  
非敏感问题可通过公开 issue 讨论：

`https://github.com/EagleFandel/metrics-agent/issues`

Do not open public issues for exploitable vulnerabilities before a fix is available.  
在漏洞修复前，请勿公开披露可利用的安全细节。

## What to Include / 报告建议包含信息

- Affected version(s) / 受影响版本
- Steps to reproduce / 复现步骤
- Expected impact / 影响范围
- Suggested fix (optional) / 建议修复方案（可选）

## Response SLA / 响应时效承诺

- Initial acknowledgement within 48 hours.  
  48 小时内确认收到报告。
- Initial triage and severity assessment within 5 business days.  
  5 个工作日内完成初步分级和评估。
- Status updates every 7 calendar days until resolution.  
  在修复完成前，每 7 天同步一次处理进展。

## Disclosure Process / 披露流程

1. Confirm the issue and classify severity.  
   确认问题并完成等级评估。
2. Develop and validate a fix.  
   开发并验证修复方案。
3. Prepare release notes and publish fix.  
   准备发布说明并发布修复版本。
4. Coordinate disclosure details after patch availability.  
   在补丁可用后协调披露细节。
